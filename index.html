HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An element is the type of content on a webpage (types of elements: meta-related, content division, and semantic elements.
  • HTML elements are composed of opening and closing tags which are represented by "< >"" angle brackets.
  • Multiple elements nested inside other elements are known as "child elements".
  • Parent elements are what houses the children elements -- in our example, the "body" was our parent element.
  • The head element can contain the following tags -- title, style, base, link, meta, ect.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are 3 ways to style a webpage using CSS -- Inline, Internal, and External.
  • Inline CSS: is when a developer wants to style an element in HTML (it is a quick and permanent way to style).
  • Internal CSS: in HTML you can add "style" into the head element to make some design changes that apply to that one page.
  • External CSS style sheets: this allows a developer to create all CSS changes in a separate file where changes can be made easier and then added to HTML after. (most common)

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • pwd: dispays the current working directory.
  • ls: lists files and folders in the current directory.
  • cd: navigates between directories in the path white cd .. will navigate you backwards one step at a time.
  • mkdir: creates a new directory.
  • touch: creates a file inside a directory.
  • git command must ALWAYS start a statement.
  • To Update your local main branch -- git checkout main -- git pull origin main.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A variable must always have a name and anything in your string must be in "".
  • When creating functions in JavaScript you must follow this path: 1. Add a variable 2. Add conditional logis 3. Incorporate arrays and for loops. 4. Complete the git flow.
  • console.log is used to output a message to the web by adding an arguement.
  • Strict equality operator (===) - this checks to see if two values are equal.
  • Array: stores a group of related data using [].
  • for loop: uses the predictible pattern of indicies to perform a task on all the items in an array by allowing a single block to be executed over and over.
  • First item of an array is ALWAYS 0
  • for loops contain three important statements -- 1. determine the starting point for our loop 2. the condition 3. what allow the array to iterate over and over.
  • x++ is a shortcut way of say x+1.
  • Commenting out code is when you add /* before and */ after.
